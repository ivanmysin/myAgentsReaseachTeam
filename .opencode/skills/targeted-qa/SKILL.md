# Purpose

Ответить на конкретный научный вопрос, используя базу статей. Найти релевантные источники, извлечь точный ответ с цитатами и оценить уровень доказательности. Подходит для фактических, концептуальных и методологических вопросов.

# When to use

Активируй этот навык когда:
- Пользователь задает конкретный вопрос: «правда ли, что...», «какой механизм...», «какие данные о...», «существует ли evidence for...»
- Нужен быстрый факт-чекинг с научным обоснованием
- Требуется оценить уровень доказательности утверждения
- Вопрос узкий и конкретный (не обзор!)

# Inputs needed

Перед началом определи (можно без уточнения у пользователя, если вопрос ясен):
- Тип вопроса: фактический / концептуальный / методологический / сравнительный
- Требуемая глубина ответа: краткий (1-2 абзаца + 3-5 ссылок) или развернутый (полстраницы + 5-15 ссылок)

# Procedure

**Шаг 1 — Классификация вопроса:**
1. Фактический: «Какая концентрация X используется для Y?» → точные данные
2. Концептуальный: «Почему гиппокамп важен для Z?» → объяснение с evidence
3. Методологический: «Как правильно измерять W?» → методические рекомендации
4. Сравнительный: «Что лучше: A или B?» → comparative analysis

**Шаг 2 — Многоканальный поиск:**
1. `semantic_search` — для концептуального соответствия
2. `db_search` (keyword) — для точных терминов
3. При необходимости: `db_search` (sql) для фильтрации
4. Цель: найти 5-20 наиболее релевантных статей

**Шаг 3 — Извлечение ответа:**
1. Для топ-статей: `read_article` (id, summary: true) — быстрое знакомство
2. Для наиболее релевантных: @reader с фокусом на конкретный вопрос
3. Извлекай ТОЛЬКО информацию, релевантную вопросу

**Шаг 4 — Формулировка ответа:**
1. Прямой ответ на вопрос (1-3 предложения)
2. Развернутое обоснование с цитатами
3. Оценка уверенности: high / moderate / low confidence
4. Противоречия (если есть)

# Output format

На РУССКОМ языке (в отличие от других навыков — это прямой диалог с пользователем):

```markdown
## Ответ: [краткий прямой ответ]

### Обоснование
[Развернутое объяснение с цитатами]

### Ключевые источники
| # | Статья | Релевантность | Ключевой finding |
|---|--------|--------------|-------------------|
| 1 | (Author et al., Year) | Прямой ответ | ... |
| 2 | (Author et al., Year) | Подтверждение | ... |

### Уровень уверенности: [High / Moderate / Low]
- High: сходятся данные из 3+ независимых групп
- Moderate: данные из 1-2 групп, или есть нюансы
- Low: косвенные данные, противоречия

### Противоречия (если есть)
- (Author1 et al., Year) vs (Author2 et al., Year): ...
```

# Quality bar (self-check)

- [ ] Прямой ответ дан в первом же предложении
- [ ] Каждое утверждение имеет ссылку на конкретную статью
- [ ] Указан уровень уверенности с обоснованием
- [ ] Отмечены противоречия между источниками (если есть)
- [ ] Ответ на русском, научный стиль, без воды
- [ ] Не выходит за рамки вопроса (не превращается в мини-обзор)
- [ ] Если данных недостаточно — честно сказано об этом

# Anti-patterns

- ❌ Начинать с общих слов вместо прямого ответа
- ❌ Превращать ответ на вопрос в полноценный обзор литературы
- ❌ Заявлять high confidence при наличии противоречий
- ❌ Отвечать на основе 1 статьи, если есть другие
- ❌ Давать ответ «по памяти» без проверки в базе
- ❌ Игнорировать методологические различия между статьями

# Examples

**Output:** «Да, ремапируют. Классические эксперименты (Muller and Kubie, 1987) показали... Однако степень ремапирования зависит от... (Colgin et al., 2008). Различают rate remapping и global remapping (Leutgeb et al., 2005)...» Уровень уверенности: High (подтверждено 10+ исследованиями). 8 источников.
