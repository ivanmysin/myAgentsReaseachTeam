# Purpose

Провести полный обзор литературы или мета-анализ по заданной теме в области нейробиологии гиппокампа. Результат — структурированный обзор с полной библиографией в стиле Chicago Author-Date.

# When to use

Активируй этот навык когда:
- Пользователь просит «обзор литературы», «meta-analysis», «review» по теме
- Задача требует анализа более 10 статей
- Нужно понять текущее состояние исследований в определенной области
- Задача типа «что известно о X в гиппокампе?»

# Inputs needed

Перед началом уточни у пользователя:
- Точная тема обзора (конкретный вопрос или область)
- Временной диапазон (если нужен)
- Критерии включения/исключения статей
- Ожидаемый объем финального текста
- Нужны ли сравнительные таблицы, графики, временные шкалы

# Procedure

**Этап 1 — Планирование и разведка:**
1. Определи ключевые концепции и синонимы для поиска
2. Используй `cluster_browse` (mode: "find") для поиска релевантных кластеров
3. Используй `cluster_browse` (mode: "list") чтобы понять общий ландшафт

**Этап 2 — Широкий поиск:**
1. Для поиска литературы используй  SKILL 'search-strategy'
2. Сохрани кандидатов в `output/search_log_review_<topic>.md`

**Этап 3 — Фильтрация по абстрактам:**
1. Прочитай абстракты топ-кандидатов через `read_article` (id, summary: true)
2. Отбери наиболее релевантные (обычно 20-60)
3. Сгруппируй по подтемам
4. Покажи shortlist пользователю с обоснованием — дождись подтверждения

**Этап 4 — Глубокое чтение:**
1. Для каждой статьи из shortlist используй субагента @reader
2. Reader создает конспект в `output/notes/<id>_<short_title>.md`
3. Запускай несколько @reader параллельно для ускорения
4. НИКОГДА не загружай полные тексты в контекст @Sophia

**Этап 5 — Синтез:**
1. Передай все конспекты субагенту @synthesizer
2. Synthesizer создает тематический обзор, сравнительные таблицы, gap analysis
3. Результат сохраняется в `output/drafts/synthesis_review_<topic>.md`

**Этап 6 — Написание:**
1. Передай синтез и конспекты субагенту @writer
2. Writer создает финальный текст обзора
3. Результат: `output/drafts/review_<topic>_<date>.md`

# Output format

Финальный обзор должен содержать:
```
# Title: [Review Topic]
## Abstract (200-300 words)
## Introduction (why this review is needed)
## Thematic Section 1
## Thematic Section 2
## ...
## Synthesis / Discussion
## Future Directions
## References (Chicago Author-Date)
```

Промежуточные артефакты:
- `output/search_log_review_<topic>.md` — лог поиска с обоснованием выбора
- `output/drafts/synthesis_review_<topic>.md` — результат синтеза

# Quality bar (self-check)

- [ ] Тема уточнена у пользователя до начала поиска
- [ ] Использованы минимум 3 разных поисковых подхода (semantic + keyword + cluster)
- [ ] Все поисковые запросы сохранены в лог
- [ ] Каждый этап воронки показан пользователю с чекпоинтом
- [ ] Каждое утверждение в финальном тексте имеет ссылку на конкретную статью
- [ ] Полный текст НИ ОДНОЙ статьи не загружался в контекст Софии (только через @reader)
- [ ] Список литературы полный и в формате Chicago Author-Date
- [ ] Отмечены противоречия между статьями и методологические ограничения

# Anti-patterns

- ❌ Начинать чтение полных текстов до фильтрации по абстрактам
- ❌ Загружать несколько полных текстов в контекст одновременно
- ❌ Пропускать чекпоинты и не спрашивать подтверждения пользователя
- ❌ Выдумывать ссылки или данные, которых нет в базе
- ❌ Игнорировать противоречия между статьями
- ❌ Писать обзор без предварительного синтеза через @synthesizer

# Examples

**Input:** «Сделай обзор литературы по роли тета-осцилляций в пространственной навигации»
**Output:** Структурированный обзор с секциями: механизмы генерации тета-ритма, связь с place cells, роль в навигации, открытые вопросы. ~30-150 источников.
